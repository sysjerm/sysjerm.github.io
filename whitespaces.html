<h1>Unicode spaces</h1>

<p>This document lists the various
space characters in <a href="../chars.html#10646">Unicode</a>.
For a description, consult chapter
<cite><a href=
"http://www.unicode.org/versions/latest/ch06.pdf"
type="application/pdf"
>6&nbsp;Writing Systems and Punctuation</a></cite>
and block description
<cite><a href=
"http://www.unicode.org/charts/PDF/U2000.pdf"
type="application/pdf">
General Punctuation
</a></cite>
in the Unicode standard. This document also lists three characters
that have no width and can thus be described as no-width spaces.
</p>

<h2>&#8220;Zero-width spaces&#8221;</h2>
<p>Previously
MONGOLIAN VOWEL SEPARATOR (U+180E)
was classified as a space character, now as formatting characters (with no width).
The characters
ZERO WIDTH SPACE (U+200B) and
ZERO WIDTH NO-BREAK SPACE (U+FEFF) were never classified
as space characters in Unicode, despite their name.

<p>ZERO WIDTH SPACE, when supported, can be used to indicate a line breaking
opportunity within a string. Similarly,
ZERO WIDTH NO-BREAK SPACE can be used between two characters to &#8200;glue&#8221;
them together, so that they no line breaking appears between them even
if normal processing rules would allow that.

<h2>Widths of space characters</h2>
<p>The characters U+2000&hellip;U+2006, when implemented in a font, usually have
the specific width defined for them, though small deviations exist.
Their widths are defined in terms of the em unit, i.e. the size of the font.
<p>The characters U+2007&hellip;U+200A and U+202F have no exact width assigned to them
in the standard, and implementations may deviate considerably even from the
suggested widths. Moreover, when concepts with the same names, such as
&#8220;thin space&#8221;, are used in publishing software, the meanings can be rather different.
For example, in InDesign, &#8220;thin space&#8221; is now 1/8 em
(i.e. 0.125 em, as opposite to the suggested 0.2 em)
and
&#8220;hair space&#8221; only 1/24 em (i.e. about 0.042 em, whereas the width of a
THIN SPACE glyph typically varies between 0.1 em and 0.2 em).

<h2>Notes on support in browsers and other software</h2>
<p>Web browsers and other programs may fail
to render all space characters according
to their definitions or descriptions.
Many commonly used fonts lack some of the space characters.
The situation has improved over the years, but caution is still
needed especially when text data may need to be transferred from
one program to another or may be viewed using different fonts.

<p>Modern browsers can usually find a glyph for a character
if <em>some</em> of the fonts in the system contain it. This does not
always take place, however,
See <cite><a href="/html/characters.html">Guide to using special characters in HTML</a></cite>.
Moreover, font substitution may cause undesired effects, since the widths
of characters vary by font.</p>

<p>The use of various space characters of specific
width, such as <small>THIN SPACE</small>,
is often an unnecessary risk.
Consider using other methods, such as the
features of a text processing program or (on Web pages) CSS properties like
<code class="nobr">padding</code>,
<code class="nobr">margin</code>,
<code class="nobr">word-spacing</code>,
and
<code class="nobr">letter-spacing</code>.
</p>

<h2 id="adj">Width adjustments</h2>

<p align=justify>In text processing, Web page display, and other contexts,
space characters are often &#8220;adjustable&#8221; in the
sense that they are presented in different widths, especially
to satisfy justification requirements. You might see this in effect in
this paragraph. Justification often just makes spaces wider, though
it may shrink them, too, especially in typesetting.</p>
<p>No-break spaces
are defined in Unicode as having the same width as spaces.
This does not specify what should happen to them in
justification.
The common practice has been to treat them
as having fixed width (in each font), which means that
in adjusted text, spaces and no-break spaces have different effects.</p>

<p>On web browsers, no-break spaces tended to be non-adjustable,
but modern browsers generally stretch them on justification.
Within
<a href="../www/justify.html">justified text on web pages</a>,
authors may have used no-break spaces instead of normal spaces
to prevent stretching (e.g., as in <code>5&amp;nbsp;m</code> instead
of <code>5&nbsp;m</code>). Due to changes in browser behavior,
it is better to use fixed-width spaces instead. Among then, the four-per-em
space
(e.g., as in <code>5&amp;#x2005;m</code>)
usually best corresponds to the width of a normal unstretched
space. However, the fixed-width spaces act as normal spaces
in line breaking, so you may wish to use some technique to
<a href="../html/space.html">prevent undesired line breaks</a>
(e.g.,
e.g., as in <code>&lt;nobr>5&amp;#x2005;m&lt;/nobr></code>).
Alternatively, consider using
<small>NARROW NO-BREAK SPACE</small>, which is generally treated
as non-stretchable in web browsers.
It might be adequate in contexts where strings belong together so that
they should not be split on two lines and could well be rendered with
decreased spacing between them, e.g. in expressions like
&#8221;10&#x202f;kg&#8221; and &#8221;C.&#x202f;S.&#x202f;Lewis&#8221;.

<p><small>The change in the treatment of no-break spaces, though
inconvenient, is consistent with changes in CSS specifications.
For example, clause
<a href="https://drafts.csswg.org/css-text-3/#spacing">7 Spacing</a>
of CSS Text Module Level 3 (Editor&#8217;s Draft 24 Jan. 2019)
defines the no-break space, but not the fixed-with spaces,
as a word-separator character, stretchable on justification.
</small></p>

<p>The Unicode standard describes the adjustment process
and the intended role of specific-width space characterss as follows:</p>

<blockquote cite="http://www.unicode.org/versions/Unicode4.0.0/ch06.pdf">
<div>The fixed-width space characters (U+2000..U+200A) are derived from
conventional (hot lead) typography. Algorithmic kerning and justification in computerized
ty­pog­ra­phy do not use these characters. However, where they are used, as, for example, in
typesetting mathematical formulae, their width is generally font-specified, and they typically
do not expand during justification. The exception is U+2009
<small>THIN SPACE</small>, which
sometimes gets adjusted.</div>
</blockquote>

<p>The <small>EM QUAD</small> character is canonical equivalent
to <small>EM SPACE</small>. The intended difference seems to be
in the code chart note for the latter:
&#8220;may scale by the condensation factor of a font&#8221;.
There is no such note for <small>EN SPACE</small>
to make it any different from <small>EN QUAD</small>.
It is not clear what &#8220;condensation factor&#8221; means here.</p>

<h2>Other notes</h2>

<p>The <small>MEDIUM MATHEMATICAL SPACE</small> character was added in Unicode
version&nbsp;4.0.</p>

<p>Regarding the non-breaking property of no-break space and other
characters, see
<cite><a href="../unicode/linebr.html"
>Unicode line breaking rules: explanations and criticism</a></cite>.</p>

<!--"http://www.microsoft.com/typography/developers/fdsspec/spaces.htm"-->
<p>Microsoft&#8217;s page <cite><a href=
"https://docs.microsoft.com/en-us/typography/develop/character-design-standards/whitespace"
>Space Characters Design Standards</a></cite> says:
&#8220;In digital fonts there are only two kinds of space characters supported by most computers, the<I> space</I> and the <I>no-break space</I>.&#8221;
This is somewhat misleading, since the support depends on fonts rather than computers, except for
no-break space support, which depends on programs.
</p>

<P>Alan Wood&#8217;s excellent Unicode resources contain a page on
the <a href=
"http://www.alanwood.net/unicode/general_punctuation.html"
>General Punctuation</a> block, with widths of space characters
illustrated graphically.</p>

<p>See also: <cite><a href="../styles/spaces.html">Styling spaces in CSS</a></cite>.</p>

<h2 id=visible>Visible spaces</h2>
<p>There are some graphic characters that can be used a symbols
for a space. Though sometimes called visible spaces, they are not
spaces at all but visible notations used to indicate the appearance of
spaces in instruction manuals and descriptions of texts.</p>
